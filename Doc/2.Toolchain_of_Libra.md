
## Toolchain of Libra
This section introduces the complete toolchain of the Libra compiler.
A plaintext C program is processed by Libra’s frontend, middle-end, and backend to generate the corresponding FHE CUDA C++ code.

### 1. Libra Frontend

Frontend of Libra is provided by the third-party library Polygeist, and the paths to `cgeist` and `polygeist-opt` are under `Security_Artifact/LibraTool/Polygeist/build/bin`.
The frontend can convert high-level C programs into plaintext MLIR.

The usage of frontend is as follows:

```bash
# 1. Convert C to initial MLIR using cgeist
"$CGEIST_BIN" "$INPUT_FILE" \
  -function=* \
  -S \
  -O0 \
  -memref-fullrank \
  -raise-scf-to-affine \
  -o "$TMP_MLIR_FILE"

# 2. Optimize MLIR using polygeist-opt
"$POLYGEIST_OPT_BIN" "$TMP_MLIR_FILE" \
  --canonicalize --cse \
  --sccp \
  --affine-loop-normalize \
  --canonicalize --cse \
  -o "$OUTPUT_FILE"
```

### 2. Libra Middle-end

After the build process completes of Libra compiler, two tools will be generated in `Security_Artifact/Libra/build/Libra/Tools`: `libra-opt` and `libra-translate`.
`libra-opt` can transform the plaintext IR generated by the frontend into efficient homomorphic encryption IR.

The usage of `libra-opt` is as follows:

```bash
"$LIBRA_OPT_BIN" \
  --verify-each \
  --mlir-print-ir-after-all \
  --debug-only=scfhe-pass \
  --auto-annotate-scfhe \
  --convert-to-scfhe \
  --canonicalize \
  --convert-to-simd \
  --debug-only=convert-to-simd \
  --canonicalize \
  --mode-select \
  --mode-select-cost-table="$COST_TABLE_PATH" \
  --debug-only=mode-select \
  --canonicalize \
  "$INPUT_FILE" \
  -o "$OUTPUT_FILE" &> "$LOG_FILE"
```

### 3. Libra Backend

`libra-translate` can convert the homomorphic IR generated by the middle-end into CUDA C++ code.

The usage of `libra-translate` is as follows:

```bash
"$LIBRA_TRANSLATE_BIN" \
  --emit-libra-backend \
  --allow-unregistered-dialect \
  --debug-only=libra-translate \
  "$INPUT_FILE" \
  -o "$OUTPUT_FILE" &> "$LOG_FILE"
```
