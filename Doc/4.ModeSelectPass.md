# Cost-Driven Mode Selection Pass

### 1. Problem Formulation
In Fully Homomorphic Encryption (FHE) compilation, the computation is modeled as a Directed Acyclic Graph (DAG) $G = (V, E)$, where $V$ represents operations and $E$ represents data dependencies.

We define the execution mode space as $\mathcal{M} = \{M_{vec}, M_{scl}\}$. The mode assignment for any node $v \in V$ is denoted by the mapping $\phi: V \rightarrow \mathcal{M}$. 

The objective is to find the optimal mapping $\phi^*$ that minimizes the total end-to-end execution latency $C_{total}$:
$$
C_{total}(\phi) = \sum_{v \in V} C_{op}(v, \phi(v)) + \sum_{(u, v) \in E} C_{cast}(\phi(u), \phi(v))
$$
Where $C_{op}$ is the base operation cost, and $C_{cast}$ is the context transition penalty (where $C_{cast} = 0$ if $\phi(u) = \phi(v)$).

### 2. Algorithm Overview
Finding the absolute optimal mode partition is NP-Hard. We propose a heuristic **Wave Iteration Algorithm** that utilizes bidirectional topological sweeps and cascade propagation to dynamically balance local operation costs and global transition penalties.

#### 2.1 Forward Pass (Top-Down)
The algorithm traverses the nodes in topological order. For each node $v$, it evaluates the forward local cost:
$$
Cost_{fwd}(v, m) = C_{op}(v, m) + \sum_{u \in pred(v)} C_{cast}(\phi(u), m)
$$
If a mode $m' \neq \phi(v)$ yields a lower cost, a mode flip is triggered: $\phi(v) \leftarrow m'$.

#### 2.2 Backward Pass (Bottom-Up)
To prevent the forward pass from falling into local optima, the algorithm performs a backward sweep in reverse topological order, evaluating the potential transition penalties imposed on downstream nodes:
$$
Cost_{bwd}(v, m) = C_{op}(v, m) + \sum_{w \in succ(v)} C_{cast}(m, \phi(w))
$$
Similarly, a mode flip is triggered if a lower-cost mode is found.

#### 2.3 Cascade Propagation
When a node $v$ flips its mode, it alters the local penalty context. The algorithm immediately uses $v$ as a source to broadcast a re-evaluation signal to its neighbors via Breadth-First Search (BFS). If neighbors also satisfy the flip condition under the new context, a **Cascade Flip** occurs, allowing subgraphs to collectively overcome high transition penalty barriers.

### 3. Algorithm Pseudo-code

```text
Algorithm 1: Wave Iteration for Mode Selection
Input: DAG G = (V, E), Initial mapping φ(v) = M_vec for all v ∈ V
Output: Optimized mapping φ*

1:  converged ← False
2:  while not converged and iterations < MAX_ITER do
3:      converged ← True
4:      
5:      // Phase 1: Forward Pass
6:      for each v in TopologicalSort(V) do
7:          m_best ← argmin_{m ∈ M} [ C_op(v, m) + Σ_{u ∈ pred(v)} C_cast(φ(u), m) ]
8:          if m_best ≠ φ(v) then
9:              φ(v) ← m_best
10:             TriggerCascadePropagation(v, UPWARD)
11:             converged ← False
12:         end if
13:     end for
14:
15:     // Phase 2: Backward Pass
16:     for each v in ReverseTopologicalSort(V) do
17:         m_best ← argmin_{m ∈ M} [ C_op(v, m) + Σ_{w ∈ succ(v)} C_cast(m, φ(w)) ]
18:         if m_best ≠ φ(v) then
19:             φ(v) ← m_best
20:             TriggerCascadePropagation(v, UPWARD_AND_DOWNWARD)
21:             converged ← False
22:             break // Restart iteration to stabilize the wave
23:         end if
24:     end for
25: end while
26: return φ
```
