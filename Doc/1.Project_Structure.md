# Project Directory Structure

## Overview

This document provides an overview of the project's directory layout and describes the purpose of each component.

```
Libra                       – Root directory of the Libra compiler project
 |- build                   – Directory for compiled binaries and build artifacts
 |- Doc                     – Project documentation and architecture designs
 |- HElib                   – Foundational FHE libraries
 |  └- FlyHE                – High-performance CUDA FHE library
 |- Libra                   – Core compiler source code
 |  |- Dialect              – Custom FHE dialects for Libra
 |  |- Target               – Backend code generation logic for CUDA
 |  |- Tools                – Libra Compiler utility passes and transformation source
 |  └- CMakeLists.txt       – Build configuration for core compiler components
 |- Script                  – Automation scripts for the compilation pipeline
 |  |- libra-mlir.sh        – Script for C to MLIR frontend conversion
 |  |- libra-opt.sh         – Script for middle-end optimization passes
 |  └- libra-translate.sh   – Translation of IR to CUDA code
 |- Tool                    – External compiler infrastructure
 |  |- llvm-project         – Base LLVM/MLIR framework
 |  └- Polygeist            – C-to-MLIR frontend transformation tool
 |- CMakeLists.txt          – Global project build configuration
 |- LICENSE                 – Project license file
 └- README.md               – Main setup and usage instructions
```


## Directory Descriptions

### **build/**
Contains compiled binary files, library archives, and intermediate build 
artifacts generated by the CMake system.

---

### **Doc/**
Documentation for the Libra project.

---

### **HElib/**
The backend Fully Homomorphic Encryption (FHE) library folder that supports 
Libra's execution.

#### **HElib/FlyHE/**
A our custom high-performance CUDA-accelerated FHE library. It provides the 
low-level implementations for SIMD-FHE and SISD-FHE scheme and their conversion.

---

### **Libra/**
The main source code of the Libra compiler based on the MLIR framework.

#### **Libra/Dialect/**
Contains the custom multi-level IR dialects and compiler passes used to transform 
high-level C programs into optimized CUDA FHE code.

- **Libra/Dialect/scfhe/**
  A high-level, scheme-agnostic FHE dialect that abstracts the data types and 
  homomorphic operations of cryptographic dialects. It can convert plaintext 
  MLIRs into homomorphically encrypted MLIRs.

- **Libra/Dialect/simd/**
  A scheme-specific dialect for SIMD-FHE schemes (e.g., CKKS). It defines the 
  data types and supported operations of SIMD-FHE, and can convert the SCFHE 
  dialect into the SIMD dialect.

- **Libra/Dialect/sisd/**
  A scheme-specific dialect for SISD-FHE schemes (e.g., TFHE). It defines the 
  data types and supported operations of SISD-FHE, and can map non-linear 
  operations to the SISD dialect.

- **Libra/Dialect/Analysis/**
  Implements the cost-aware analysis passes. These passes estimate computational 
  costs on GPGPU hardware to guide the dynamic selection of the most efficient 
  FHE scheme. Based on cost analysis, it can choose schemes for FHE operators and 
  dynamically adjust the conversion boundary between SIMD and SISD.

- **Libra/Dialect/Parallel/**
  Contains GPU-aware dialect logic used to add parallelism and synchronization 
  tokens to the computation graph. It can label a program’s parallel and serial 
  execution, bridging the gap between FHE algorithms and GPU execution models.

#### **Libra/Target/**
Implements the backend lowering logic. It defines the interface to the backend FHE 
execution library and is responsible for lowering the final optimized IR into 
CUDA C++ code compatible with the library.

#### **Libra/Tools/**
Source code for internal compiler utilities and command-line interfaces used for optimizing and translating IR.

---

### **Script/**
Automation scripts for managing the multi-stage Libra compilation workflow.

- **libra-mlir.sh**
  Frontend script that invokes the conversion of high-level source code into initial MLIR dialects.

- **libra-opt.sh**
  Middle-end script that executes Libra’s optimization passes, including segmentation and cost-aware scheme selection.

- **libra-translate.sh**
  Backend scripts that perform the final translation of optimized IR into CUDA C++ source code.

---

### **Tool/**
External third-party infrastructures and tools that serve as the foundation for the Libra compiler.

- **llvm-project/**
  The base LLVM and MLIR compiler infrastructure.

- **Polygeist/**
  The C-to-MLIR frontend tool used for initial program ingestion and semantic preservation.

---
